FPGA图像处理-FAST特征点检测

代码GitHub链接

FAST特征点检测基本步骤

算法思想：如果一个像素与它邻域的像素差别较大（过亮或过暗）, 那它更可能是特征点。

算法步骤：

  • 上图所示，一个以像素p为中心，半径为3的离散化Bresenham圆上，取16个邻域像素点point1-point16；

  • 定义一个阈值。计算point1、point13与中心p的像素差，若它们绝对值都小于阈值，则p点不可能是特征点，反之p当做候选点；

  • 若p是候选点，则计算point1、point9、point5、point13与中心p的像素差，若它们的绝对值有至少3个超过阈值，则当做候选点进行下一步判断；

  • 若p是候选点，则计算p1到p16这16个点与中心p的像素差，若它们至少有连续9个超过阈值，则是特征点；

  • 对图像进行非极大值抑制：计算特征点出的FAST的score值（16个点与中心差值的绝对值总和），判断以特征点p为中心的一个邻域（如3x3或5x5）内，p点是否为score值最大的点，若p是邻域所有特征点中score值最大的，则保留；否则，抑制。若邻域内只有一个特征点，则保留。

FAST特征点检测的rtl设计

系统框图如下，可以分为Bresenham_16point_8bit模块，Get_Feature_point模块、Get_score模块和NMS模块。在FAST_top模块中对上述模块进行例化调用。

Bresenham_16point_8bit模块

Bresenham_16point_8bit模块用于生成中心点和对应Bresenham邻域16点的灰度值补码。模块中设计了7个移位寄存器对输入像素点进行缓存，移位寄存器的深度为每列像素点个数*像素点数据位宽。邻域中的每个点在移位寄存器中的对应位置如下图：

由于算法计算中包含对邻域点与中心点相减求绝对值的操作，因此需要将灰度值数据转换为补码，输出的point1-point16和center灰度值为9bit。其中point1-point16低8位与输入保持不变，最高位补零，center低8位按位取反加1，最高位补1。
当第一个像素点处于模板中心位置时，消耗的时钟周期为4*数据每列个数-3，因此需要将同步信号和数据有效信号延迟对应周期。

Get_Feature_poin模块

根据FAST特征点的定义判定一个检测点是否为特征点，需要依次查看周围16个点的连续9个点与中心的差是否大于阈值，在OpenCV中，作者使用的方法是优先判断上下左右四个点，即P1, P5, P9, P13，并且引入了阈值列表来判断。由于图像中特征点占像素点的少部分，该方法针对CPU的逐点计算可以起到非常好的优化效果，但是对于FPGA或GPU的并行计算，大量的if-else语句反而并不适用。
在Get_Feature_poin模块设计中，本模块采用了阈值标志位threshold_flag[15:0]作为判断依据。比较补码运算的结果与设定阈值判断标志位，并且将16种至少有连续9点大于阈值的情况存入寄存器，仅需比较寄存器与阈值标志位寄存器的与运算结果即可判断连续9点满足阈值的情况。

  • 第一个周期中，将point1-point16的灰度值补码与center的灰度值补码相加，即point1-point16与中心做差；

  • 第二个周期中，取做差后的绝对值，若符号位为0，则取低8位，若为1则对低8位取反并加1

  • 第三个周期比较做差后的绝对值与阈值的大小，大于阈值对应的threshold_flag标志位置1，反之置0；

  • 第四个周期根据标志位判断是否为特征点，输出特征点有效信号。

由于第二个周期中计算出的绝对值在第四个周期也被用到，因此需要将绝对值缓存一个周期。
为了判断像素点对应的坐标，需要设置两个计数器height_cnt和width_cnt。项目中数据为按列输入，要求height_cnt在时钟信号激励下计数，计数最大值为图像行数，width_cnt在height_cnt计数满时加1，最大值为图像列数。为了保证坐标计数的准确，需要与第四个周期的电路进行同步，因此这两个计数器需要在输入数据有效信号延迟3个周期后开始计数。

Get_score模块

在OpenCV中，对特征点打分方法是改变阈值threshold，找到最大的一个threshold使该点恰好满足特征点判定的条件，即满足n个以上连续的点满足与中心的差大于该阈值。这个threshold即为该点的得分值score = threshold。如果图像的像素是uint类型，score一定为圆周上某个与特征点差值的绝对值减一。该方法在OpenCV中实现需要遍历16个邻域点。

在Get_score模块设计中，采用了大量并行计算来提高速度。由于score值一定为邻域点与中心点之差中的某一个，因此第一步将每个点与中心的的差作为阈值，按照Get_Feature_poin模块的方法判断是否满足特征点条件，并且设置一个标志寄存器S[15:0]，满足条件对应位置1反之置0。第二步需要在这些能够满足条件的点中找出最大值即为score，本模块设计了如图所示的比较结构，对这些差值乘以标志位后两两比较。

对于非特征点和边界点，score的值都为0。

NMS模块

NMS模块实现对特征点进行非极大值抑制。在本设计中取3*3区域内的最大值作为非极大值抑制的输出。

首先使用matrix_3x3_11bit模块提取33模板。该模块包含3个深度为图像行数11bit的首尾相连的寄存器，对同步信号和有效信号的延迟为2*图像行数-1。

当中心score值大于8个邻域时，输出xy坐标，以及一个坐标有效信号。坐标计数器设计同get_score模块。

test bench与仿真结果

输入图像来自读取bin文件生成数据流，输出xy坐标和每个像素点score值到bin文件。测试结果对比matlab计算结果，计算的特征点数量、坐标与matlab结果绝大部分一致。

`timescale 1ns / 1ns
module tb_FAST();

    reg ACLK_in;
    reg ARESTN_in;
    reg [7:0] TDATA_in;
    reg TSTRB_in;
    reg TLAST_in;
    reg TVALID_in;
    reg TUSER_in;
    wire TREADY_out;
    
   wire ACLK_out;     
   wire ARESTN_out;    
   wire [7:0] TDATA_out; 
   wire TSTRB_out;      
   wire TLAST_out;       
   wire TVALID_out;      
   wire TUSER_out;       
   reg TREADY_in;       
  reg FAST_EN;
  reg [10:0] width_in;
  reg [10:0] height_in;
  reg [7:0] threshold;
    
    reg [7:0] temp8b;
    integer fd1;
    integer stop_flag;
    integer pixel_cnt;
    integer V_cnt;
    integer H_cnt;
    integer V_next;
    integer start_num, pause_num;

    parameter period = 500;
    parameter data_depth = 8;


    FAST_AXI #(
    .data_depth(data_depth)
    )FAST_AXI_init(
    .ACLK_in(ACLK_in),
    .ARESTN_in(ARESTN_in),
    .TDATA_in(TDATA_in),
    .TSTRB_in(TSTRB_in),
    .TLAST_in(TLAST_in),
    .TVALID_in(TVALID_in),
    .TUSER_in(TUSER_in),
    .TREADY_out(TREADY_out),
    
    .FAST_EN(FAST_EN),  
    .width_in(width_in),
    .height_in(height_in),
    .threshold(threshold),
    
    .ACLK_out(ACLK_out),      
    .ARESTN_out(ARESTN_out),   
    .TDATA_out(TDATA_out),
    .TSTRB_out(TSTRB_out),   
    .TLAST_out(TLAST_out),   
    .TVALID_out(TVALID_out),   
    .TUSER_out(TUSER_out),   
    .TREADY_in(TREADY_in)   
    );

    initial
    begin
    FAST_EN = 1;
    height_in = 720;
    width_in = 1280;
    threshold = 60;
    ARESTN_in = 0;
    TVALID_in = 0;
    TREADY_in = 1;
    #(period);
    ARESTN_in = 1;
    #(period);
    TVALID_in = 1;
    #(period);
    while(pixel_cnt < 32'd921600)
    begin
    start_num = {$random}%721 + 2;
    pause_num = {$random}%11 + 2;
    while(start_num > 1)
    begin
    start_num = start_num - 1;
    #(period);
    end
    while(pause_num > 1)
    begin
    pause_num = pause_num - 1;
    TVALID_in = 0;
    #(period);
    end
    TVALID_in = 1;
    end
    TVALID_in = 0;
    end

    //读
    initial
    begin
    stop_flag = 0;
    pixel_cnt = 0;
    V_next = 0;
    V_cnt = 1;
    H_cnt = 1;
    fd1 = $fopen("E:/my_verilog/prev.bin", "rb");
    #(3*period);

    while(pixel_cnt < 32'd921600)
    begin
    if(TREADY_out == 1'b1 && TVALID_in == 1'b1)
    begin
    $fread(temp8b, fd1, , 1);
    TDATA_in = temp8b;
    pixel_cnt = pixel_cnt + 1;

    if(V_cnt == 1 && V_next == 0)
    begin
    TUSER_in = 1;
    V_next = 1;
    end
    else if(H_cnt == 720)
    TLAST_in = 1;
    else;

    if(V_cnt < 1280)
    V_cnt = V_cnt + 1;
    else
    V_cnt = 1;

    if(H_cnt < 720)
    H_cnt = H_cnt + 1;
    else
    H_cnt = 1;

    #(period/2);
    TUSER_in = 0;
    TLAST_in = 0;
    #(period/2);
    end
    else
    begin
    pixel_cnt = pixel_cnt;
    H_cnt = H_cnt;
    V_cnt = V_cnt;
    #(period);
    end
    if(pixel_cnt == 32'd921600)
    V_next = 0;
    end
    $fclose(fd1);
    stop_flag = 1;
    pixel_cnt = pixel_cnt + 1;
    TVALID_in = 0;
    #(period);
    //		$stop;
    end

    //写
    integer fd2;
    initial
    begin
    fd2 = $fopen("E:/my_verilog/AXI/FAST/FAST/score.bin", "wb");

    while(1)
    begin
    if(TVALID_out == 1)
    begin
    $fwrite(fd2,"%02x",TDATA_out);
    end
    else if(TVALID_out == 0 && stop_flag == 1)
    begin
    $fclose(fd2);
    $stop;
    end
    #(period);
    end
    end



    initial
    begin
    ACLK_in = 0;
    forever #(period/2) ACLK_in = ~ACLK_in;
    end

endmodule
